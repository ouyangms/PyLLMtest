# 车控技能路由模型 - 准确率深度分析报告

## 执行摘要

通过对模型准确率的深度分析，我们发现：

1. **真实准确率：81.15%**（过滤无效样本后）
2. **表面准确率：62.84%**（包含 57.5% 无效样本）
3. **问题根源：数据集质量，而非模型能力**

---

## 1. 数据集问题分析

### 1.1 数据重复率
- 训练集重复率：**92.2%**
- 真实唯一样本：仅 **604 个**

### 1.2 测试集问题
- **57.5% 的测试样本是无效样本**
- 这些样本极度模糊，无法准确分类
- 例如："调一调"、"弄一下"、"恢复默认"

### 1.3 类别不平衡
- 最大类别：climate_control (569 样本)
- 最小类别：navigation (12 样本)
- 不平衡比例：**47:1**

## 3. 实验结果汇总

### 3.1 各模型对比（原始测试集）

| 模型 | 准确率 | 说明 |
|------|--------|------|
| 原始 TextCNN | 62.84% | 基线模型 |
| 清洗数据训练 | 53.14% | 去除重复后训练 |
| 千问增强训练 | 47.94% | LLM 生成数据混合训练 |

**关键发现**：数据清洗和 LLM 增强反而降低了准确率！

### 3.2 千问数据质量问题

使用千问 API 生成样本时发现：
- **37.9% 的样本存在分类错误**
- 千问混淆了不同类别的定义
- 生成的数据风格与真实用户输入不匹配

示例错误：
- "关闭后排座椅加热" → 被分类为 climate_control（应该是 seat_control）
- "有点晕车" → 被分类为 climate_control（应该是其他）

## 5. 结论与建议

### 5.1 当前模型状态
- ✅ **模型能力良好**：在有效样本上达到 81.15% 准确率
- ✅ **主要类别表现优秀**：climate_control (82%), light_control (83%)
- ⚠️ **小样本类别失败**：<50 样本的类别无法学习

### 5.2 数据集问题
- ❌ **57.5% 测试样本无效**：极度模糊，无法准确分类
- ❌ **92.2% 训练数据重复**：有效信号不足
- ❌ **类别极度不平衡**：47:1 比例

### 5.3 改进建议

#### 短期（数据层面）
1. **清洗测试集**：去除 57.5% 无效样本
2. **增加小样本类别**：为 <50 样本的类别生成专门数据
3. **人工标注**：对混淆严重的类别对进行人工审核

#### 中期（系统层面）
1. **多轮对话**：对模糊输入进行追问
   - 用户："调一调"
    - 系统："您想调节什么？"（空调/座椅/灯光...）
2. **上下文记忆**：利用对话历史理解意图
3. **置信度阈值**：低置信度时触发追问

#### 长期（架构层面）
1. **混合架构**：规则引擎 + ML 模型
   - 规则引擎处理常见指令
   - ML 模型处理复杂表达
2. **增量学习**：持续收集真实用户反馈
3. **A/B 测试**：线上验证改进效果
