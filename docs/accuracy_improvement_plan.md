# TextCNN 准确率提升方案

## 当前状态

- **准确率**: 62.84%
- **目标**: >90%
- **差距**: +27.16%

## 数据分析发现

### 问题1: 极度类别不平衡 ⚠️
```
climate_control:  2,653 样本 (34.4%) ← 最大
navigation:           51 样本 (0.7%)  ← 最小
不平衡比例: 52:1
```

**7个少数类别**（样本 <5%）：
- navigation (51)
- phone_call (161)
- mirror_control (249)
- door_control (118)
- vehicle_info (307)
- driving_assist (190)
- charging_energy (128)

### 问题2: 数据质量问题 ⚠️
- **重复率 92.2%**
- 只有 604 个唯一文本
- 大量简单重复指令："开一点"、"关一点"、"调大一点"

### 问题3: 样本不足 ⚠️
- navigation 仅 51 样本
- 总训练样本 7,722 个
- 平均每个类别 594 个（但分布极不均）

### 问题4: 类别混淆
- 多数类别共享相同关键词
- 例：climate_control ↔ driving_assist 词汇重叠 100%

---

## 改进方案

### 方案1: 数据增强 (优先级: 高) 🎯

**1.1 LLM 泛化生成**
- 使用 Qwen/Gemma 生成多样化训练样本
- 为每个类别生成至少 100 个高质量样本
- 5维度模板：
  - 标准指令型：清晰完整的表达
  - 场景/感受型：描述状态（"太热了"、"有点冷"）
  - 模糊/口语型：省略主语、方言词、极简表达
  - 否定/反向型：边界情况
  - 组合/上下文型：多轮对话模拟

**1.2 同义词替换**
- 使用预定义同义词典
- 随机替换关键词

**1.3 回译增强**
- 中文 → 英文 → 中文
- 生成句式不同的等价表达

**预期收益**: +15-20% 准确率

---

### 方案2: 模型改进 (优先级: 高) 🎯

**2.1 增加模型容量**
```python
# 当前: TextCNNLite
embedding_dim: 64 → 128
num_filters: 128 → 256
filter_sizes: [2,3,4] → [2,3,4,5,6]
```

**2.2 添加注意力机制**
- 字符级自注意力
- 增强关键特征权重

**2.3 预训练嵌入**
- 使用 word2vec/fastText 字符嵌入
- 或使用 BERT 提取字符级嵌入作为特征

**预期收益**: +8-12% 准确率

---

### 方案3: 训练优化 (优先级: 中)

**3.1 类别权重平衡**
```python
# 计算类别权重
class_weights = 1.0 / (class_counts / total_samples)
criterion = nn.CrossEntropyLoss(weight=class_weights)
```

**3.2 Focal Loss**
- 关注难分类样本
- 减少简单样本的权重

**3.3 更长训练**
- 当前早停: 10 epoch
- 增加 warmup + cosine annealing

**预期收益**: +5-8% 准确率

---

### 方案4: 集成学习 (优先级: 低)

**4.1 多模型投票**
- 训练 5 个不同初始化的模型
- 软投票取平均

**4.2 Bagging**
- 不同数据子集训练
- 模型融合

**预期收益**: +3-5% 准确率
